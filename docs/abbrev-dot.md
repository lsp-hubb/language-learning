# 英文句点误判问题

## 问题描述

`findSentenceBoundaries()` 用于按 `.` 分割英文句子，但英文中存在大量非句号用途的 `.`：

- **缩写点**：`Mr.` `Dr.` `Jr.` `U.S.` `a.m.` `etc.`
- **小数点**：`3.14` `0.5`
- **省略号**：`...`

## 影响范围

句子分割结果直接影响**翻译句子高亮**和 **r 键长难句标注**的索引对齐（英文第 n 句 ↔ 中文第 n 句）。

## 判定规则

只将满足以下条件的 `.` 视为句子边界：

```
后面是 → 空格 / 引号("') / 右括号 / 行尾 / 换行
+ 且 "." 前的单词不是常见缩写
```

### 缩写白名单

```js
mr, mrs, ms, dr, st, jr, sr, vs,
etc, inc, ltd, ave, dept, est, govt,
jan, feb, mar, apr, jun, jul, aug, sep, oct, nov, dec
```

+ **单字母缩写**统一跳过（`U.S.`、`a.m.`、`p.m.`）

### 示例对照

| 文本 | 判定 | 原因 |
|------|------|------|
| `killed."` | ✅ 句号 | 后面引号，单词正常 |
| `ended. Next` | ✅ 句号 | 后面空格，`ed` 不在白名单 |
| `Jr., was` | ❌ 跳过 | 后面逗号 |
| `Mr. Smith` | ❌ 跳过 | `mr` 在白名单 |
| `U.S. ` | ❌ 跳过 | `S` 为单字母 |
| `etc. ` | ❌ 跳过 | `etc` 在白名单 |
| `3.14` | ❌ 跳过 | 后面是数字 |
| `...` | ❌ 跳过 | 后面是 `.` |
| `a.m. ` | ❌ 跳过 | `m` 为单字母 |

## 局限

- 白名单可能遗漏某些领域特有缩写（如法律引文 `v.`、学衔 `Ph.D.`）
- 新缩写需手动添加至 `ABBREV_SET`
- 完美分割需要 NLP 句点消歧，当前启发式规则覆盖常见场景
